### 更新规范

#### 白皮书更新（public/whitepaper/）

**单步流程：**
- 直接更新 `public/whitepaper/` 对应 Markdown 文件
- 如新增章节，同步更新 `chapters.json`
- 无需同步到 steering

#### 开发规范更新（docs/standards/）

**两步同步流程：**

1. **更新完整版** `docs/standards/xx.xxx-xx.xxx.md`
   - 包含完整说明、示例、原理
   - 人类可读的详细文档

2. **同步精简版** `.kiro/steering/xx.xxx-xx.xxx.md`
   - 提取核心规则
   - 顶部添加完整版引用
   - AI 自动加载

#### 新增规范文档

1. 确定大类编号（00-09）
2. 按命名格式创建完整版
3. 创建同名精简版
4. 更新本文件的文件对应关系表

### 精简版格式要求

每个精简版文件顶部必须包含完整版引用：

```markdown
# 标题（精简版）

> 完整版: `docs/standards/xx.xxx-xx.xxx.md`

[精简内容...]
```
