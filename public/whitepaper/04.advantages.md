# 核心优势

## 差异化竞争力

轻养到家相比现有平台，具有以下核心优势：

## 1. 技师借调机制

### 痛点解决

传统模式下，商家面临：
- 旺季缺人，订单流失
- 淡季养人，成本高企
- 临时招人，质量难保

### 我们的方案

```
┌─────────────────────────────────────────────────────┐
│                   技师借调流程                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   店铺A（缺人）          平台           店铺B（有余）  │
│   ┌─────────┐        ┌─────────┐      ┌─────────┐  │
│   │发起借调 │───────►│智能匹配 │◄─────│技师空闲 │  │
│   │  需求   │        │  撮合   │      │  登记   │  │
│   └────┬────┘        └────┬────┘      └────┬────┘  │
│        │                  │                │       │
│        │    ┌─────────────┼─────────────┐  │       │
│        ▼    ▼             ▼             ▼  ▼       │
│   ┌─────────────────────────────────────────────┐  │
│   │              技师完成服务                     │  │
│   │  • 店铺A支付服务费                           │  │
│   │  • 店铺B获得分成                             │  │
│   │  • 技师获得工资+补贴                         │  │
│   │  • 平台收取撮合费                            │  │
│   └─────────────────────────────────────────────┘  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 各方收益

| 角色 | 收益 |
|------|------|
| 借入店铺 | 解决用人荒，不流失订单 |
| 借出店铺 | 闲置资源变现，额外收入 |
| 技师 | 更多工作机会，收入增加 |
| 平台 | 撮合服务费，提升粘性 |

## 2. 评价可迁移

### 痛点解决

传统模式下，技师的评价：
- 绑定在店铺，换店清零
- 被店铺控制，可能被删除
- 不够全面，只有星级

### 我们的方案

**评价跟着技师走，不被店铺绑架**

| 特性 | 说明 |
|------|------|
| 永久保留 | 评价存储在平台，不随店铺变化 |
| 多维度 | 手法、态度、准时、沟通、卫生 |
| 不可删除 | 真实评价，店铺无法干预 |
| 可追溯 | 顾客可查看技师完整历史 |

### 评价维度

```
技师评分卡
├── 专业技能 ★★★★★
│   └── 手法熟练度、力度控制、穴位准确
├── 服务态度 ★★★★☆
│   └── 礼貌程度、沟通能力、耐心程度
├── 准时守约 ★★★★★
│   └── 是否准时到达、服务时长是否足够
├── 沟通能力 ★★★★☆
│   └── 需求理解、过程沟通、建议专业性
└── 卫生习惯 ★★★★★
    └── 个人卫生、工具清洁、环境整理
```

## 3. T+N 结算机制

### 痛点解决

技师常见遭遇：
- 工资拖欠，讨薪困难
- 店铺跑路，血本无归
- 结算不透明，被克扣

### 我们的方案

**平台托管资金，保障技师权益**

```
订单完成 ──► 资金进入平台托管 ──► T+N天后自动结算
                    │
                    ▼
            ┌───────────────┐
            │ 资金安全保障   │
            │ • 银行托管账户 │
            │ • 自动结算     │
            │ • 不可挪用     │
            └───────────────┘
```

| 结算周期 | 适用场景 | 说明 |
|---------|---------|------|
| T+1 | 散技师 | 次日结算，快速到账 |
| T+7 | 店铺技师 | 周结，便于店铺管理 |
| T+15 | 商户 | 半月结，资金周转 |

### 保障机制

- **资金托管**：订单资金由银行托管，平台不可挪用
- **自动结算**：到期自动打款，无需人工干预
- **争议处理**：有争议时资金冻结，平台介入仲裁

## 4. 散技师生态

### 什么是散技师？

不隶属于任何店铺，独立接单的技师。

### 平台支持

| 支持项 | 说明 |
|--------|------|
| 独立接单 | 直接接受用户订单 |
| 自主定价 | 自己设定服务价格 |
| 灵活时间 | 自己安排工作时间 |
| 评价积累 | 建立个人品牌 |
| 保险保障 | 平台提供意外险 |

### 与店铺技师对比

| 维度 | 店铺技师 | 散技师 |
|------|---------|--------|
| 收入 | 底薪+提成 | 纯提成，上限更高 |
| 自由度 | 受店铺约束 | 完全自主 |
| 稳定性 | 相对稳定 | 波动较大 |
| 适合人群 | 求稳定 | 求高收入、自由 |
